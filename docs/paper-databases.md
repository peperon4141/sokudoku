# 論文にアクセスできる主要なサイト

## 1. 論文データベースの種類

### 管理サイトは複数ある
- **はい、複数の管理サイトがあります**
- 各サイトは異なる特徴と利用条件を持っています
- 無料でアクセスできるサイトと有料のサイトがあります

## 2. 日本の論文データベース

### J-STAGE（科学技術情報発信・流通総合システム）
- **URL**: https://www.jstage.jst.go.jp/
- **説明**: 日本の学術論文を公開するプラットフォーム
- **特徴**:
  - 日本語の論文が豊富
  - オープンアクセスの論文が多い
  - 多くがCC BYライセンス
  - 無料でアクセス可能
- **検索**: キーワード、著者名、発行年、分野で検索可能
- **API**: 公式APIが提供されている場合がある
- **利用条件**: 各論文のライセンスを確認（多くがCC BY）

### CiNii（サイニィ）
- **URL**: https://ci.nii.ac.jp/
- **説明**: 日本の学術論文情報を検索できるデータベース
- **特徴**:
  - 日本の大学や研究機関の論文を検索
  - 全文が公開されている場合と要約のみの場合がある
  - 無料で検索可能
- **検索**: キーワード、著者名、機関名で検索可能
- **利用条件**: 各論文のライセンスを確認

### 国立国会図書館デジタルコレクション
- **URL**: https://dl.ndl.go.jp/
- **説明**: 日本の古い論文や資料をデジタル化
- **特徴**:
  - 主に古い論文（著作権が消滅したもの）
  - 利用規約を確認する必要がある
- **利用条件**: 利用規約を確認

## 3. 国際的な論文データベース

### arXiv（アーカイブ）
- **URL**: https://arxiv.org/
- **説明**: 物理学、数学、コンピュータサイエンスなどの論文を公開
- **特徴**:
  - 主に英語
  - プレプリント（査読前の論文）が多い
  - 無料でアクセス可能
  - 多くがCCライセンス
- **API**: 公式APIが提供されている
- **利用条件**: 各論文のライセンスを確認

### PubMed
- **URL**: https://pubmed.ncbi.nlm.nih.gov/
- **説明**: 医学・生命科学の論文を検索
- **特徴**:
  - 主に英語
  - 医学・生命科学の論文が豊富
  - 無料で検索可能
  - 全文が公開されている場合と要約のみの場合がある
- **API**: 公式APIが提供されている
- **利用条件**: 各論文のライセンスを確認

### Google Scholar（グーグルスカラー）
- **URL**: https://scholar.google.com/
- **説明**: 学術論文を検索できるGoogleのサービス
- **特徴**:
  - 様々なデータベースから論文を検索
  - **J-STAGEやCiNiiも検索対象に含まれている**
    - **J-STAGE**: 2006年から検索対象となっている
    - **CiNii**: 2007年4月から検索対象となっている（約300万件の論文データ）
  - 無料で検索可能
  - 全文が公開されている場合と要約のみの場合がある
- **API**: 公式APIは提供されていない（非公式のAPIがある）
- **利用条件**: 
  - **個人利用（読むだけ）**: 問題なし
  - **商用利用**: 各論文のライセンスを確認する必要がある
  - **注意**: Google Scholar自体は検索エンジンであり、論文の著作権は各論文の著作者が保有

### DOAJ（Directory of Open Access Journals）
- **URL**: https://doaj.org/
- **説明**: オープンアクセスの学術雑誌を検索
- **特徴**:
  - オープンアクセスの論文のみ
  - 無料でアクセス可能
  - 多くがCCライセンス
- **利用条件**: 各論文のライセンスを確認

### IEEE Xplore
- **URL**: https://ieeexplore.ieee.org/
- **説明**: 電気・電子工学、コンピュータサイエンスの論文
- **特徴**:
  - 主に英語
  - 有料の論文が多い
  - 一部の論文はオープンアクセス
- **利用条件**: 各論文のライセンスを確認

### ACM Digital Library
- **URL**: https://dl.acm.org/
- **説明**: コンピュータサイエンスの論文
- **特徴**:
  - 主に英語
  - 有料の論文が多い
  - 一部の論文はオープンアクセス
- **利用条件**: 各論文のライセンスを確認

## 4. Google Scholarについて

### 使用できるか？
- ✅ **個人利用（読むだけ）**: 問題なし
- ⚠️ **商用利用**: 各論文のライセンスを確認する必要がある
- ❌ **API**: 公式APIは提供されていない（スクレイピングは利用規約に違反する可能性がある）

### Google Scholarの特徴
- **検索エンジン**: Google Scholar自体は検索エンジンであり、論文の著作権は各論文の著作者が保有
- **J-STAGEとCiNiiも検索対象**: 
  - **J-STAGE**: 2006年から検索対象となっている（科学技術振興機構の電子ジャーナル）
  - **CiNii**: 2007年4月から検索対象となっている（国立情報学研究所の約300万件の論文データ）
  - これらの連携により、Google ScholarからJ-STAGEやCiNiiに収録されている日本の学術論文を検索・閲覧できる
- **無料で検索可能**: 誰でも無料で論文を検索できる
- **全文へのリンク**: 論文の全文が公開されている場合、リンクを提供
- **要約のみ**: 全文が公開されていない場合、要約のみが表示される
- **有料論文**: 全文が有料の論文も検索結果に表示される（[PDF]や[HTML]のリンクがない）

### 使用可能か？
- ✅ **個人利用（読むだけ）**: 問題なし
- ⚠️ **商用利用**: 各論文のライセンスを確認する必要がある

### 特徴
- **検索エンジン**: Google Scholar自体は検索エンジンであり、論文の著作権は各論文の著作者が保有
- **全文へのリンク**: 論文の全文が公開されている場合、リンクを提供
- **要約のみ**: 全文が公開されていない場合、要約のみが表示される

### 注意点
- **著作権**: Google Scholarで見つけた論文でも、著作権は各論文の著作者が保有
- **ライセンス確認**: 商用利用する場合は、各論文のライセンスを確認する必要がある
- **API**: 公式APIは提供されていないため、スクレイピングは利用規約に違反する可能性がある

## 7. 「注目」の定義と測定方法

### 各データベースが「注目」をどう定義しているか

「注目」や「人気」の定義は、データベースやサービスによって異なります。一般的に使用される指標は以下の通りです：

#### 1. 被引用数（Citation Count）
- **定義**: 他の論文からどれだけ引用されているかを示す数値
- **測定方法**: 各データベースが独自に収集・集計
- **特徴**:
  - 論文の影響力を測る基本的な指標
  - 時間が経つほど増加する傾向がある（古い論文ほど引用されやすい）
  - 分野によって引用数が大きく異なる（医学・生命科学は高め、人文科学は低め）
- **提供しているデータベース**:
  - OpenAlex: `cited_by_count`フィールドで提供
  - Web of Science: 引用数情報を提供
  - Scopus: 引用数情報を提供
  - Google Scholar: 引用数情報を表示（ただし、APIで取得不可）

#### 2. ダウンロード数（Download Count）
- **定義**: 論文がどれだけダウンロードされたかを示す数値
- **測定方法**: 各プラットフォームが独自に集計
- **特徴**:
  - 論文の即時的な関心度を示す
  - 被引用数よりも早く反映される
  - プラットフォームによって大きく異なる
- **提供しているデータベース**:
  - arXiv: ダウンロード統計を提供（ただし、APIでは取得不可）
  - J-STAGE: 一部の論文でダウンロード数を表示

#### 3. 閲覧数（View Count）
- **定義**: 論文ページがどれだけ閲覧されたかを示す数値
- **測定方法**: 各プラットフォームが独自に集計
- **特徴**:
  - 論文の関心度を示す最も即時的な指標
  - ダウンロード数よりも多い傾向がある
- **提供しているデータベース**:
  - arXiv: 閲覧統計を提供（ただし、APIでは取得不可）
  - J-STAGE: 一部の論文で閲覧数を表示

#### 4. Altmetrics（代替指標）
- **定義**: 従来の引用数以外の指標を用いて論文の影響力を評価
- **測定方法**:
  - SNSでの言及数（Twitter、Facebook、LinkedInなど）
  - ニュース記事での取り上げられ方
  - ブログでの言及数
  - Wikipediaでの引用数
  - 政策文書での引用数
- **特徴**:
  - 学術界以外での影響力を測定
  - 即時的な反応を捉えやすい
  - 分野によって大きく異なる
- **提供しているデータベース**:
  - Altmetric.com: 専用のAltmetricsサービス
  - 一部のジャーナル: Altmetricsを表示

#### 5. h-index（エイチインデックス）
- **定義**: 研究者や論文の生産性と影響力を同時に評価する指標
- **測定方法**: 特定の数以上引用された論文数を示す
- **特徴**:
  - 研究者の評価に使用されることが多い
  - 個別の論文の評価には使用されない
- **提供しているデータベース**:
  - Web of Science: h-indexを提供
  - Scopus: h-indexを提供

#### 6. インパクトファクター（Impact Factor）
- **定義**: 特定のジャーナルが過去数年間に受けた平均被引用数を示す指標
- **測定方法**: ジャーナル全体の影響力を評価
- **特徴**:
  - ジャーナルの評価に使用される
  - 個別の論文の評価には使用されない
- **提供しているデータベース**:
  - Journal Citation Reports (JCR): インパクトファクターを提供

### データベース別の「注目」の定義

#### OpenAlex
- **主な指標**: `cited_by_count`（被引用数）
- **ソート方法**: `sort=cited_by_count:desc`で引用数の多い順にソート可能
- **その他の指標**: `publication_date`（出版日）、`relevance_score`（関連度スコア）

#### arXiv
- **主な指標**: ダウンロード数、閲覧数（ただし、APIでは取得不可）
- **ソート方法**: 
  - `sortBy=submittedDate`: 投稿日順
  - `sortBy=lastUpdatedDate`: 最終更新日順
  - `sortBy=relevance`: 関連度順
- **注意**: 引用数やダウンロード数はAPIでは取得できない

#### PubMed
- **主な指標**: 被引用数（一部の論文のみ）
- **ソート方法**: 
  - `sort=pub_date`: 出版日順
  - `sort=relevance`: 関連度順
- **注意**: 引用数情報は限定的

#### J-STAGE
- **主な指標**: ダウンロード数、閲覧数（一部の論文のみ）
- **ソート方法**: 検索結果のソート機能は限定的
- **注意**: APIでは統計情報の取得が難しい

### 「注目」の定義の課題

1. **分野による違い**: 医学・生命科学は引用数が高め、人文科学は低め
2. **時間による違い**: 古い論文ほど引用されやすい
3. **プラットフォームによる違い**: 各データベースが独自に集計しているため、数値が異なる
4. **指標の限界**: 引用数が多いからといって、必ずしも「注目」されているとは限らない

## 8. 網羅的な検索方法

### 複数のデータベースを組み合わせる

各データベースは特定の分野や地域に特化しているため、複数のデータベースを組み合わせて利用することで、より広範な論文を検索できます。

#### 推奨される組み合わせ

1. **日本の論文を網羅的に検索**:
   - J-STAGE（日本の学術論文）
   - CiNii（日本の学術論文）
   - Google Scholar（J-STAGEとCiNiiも検索対象）

2. **国際的な論文を網羅的に検索**:
   - OpenAlex（完全にオープンな学術知識グラフ）
   - arXiv（プレプリント）
   - PubMed（医学・生命科学）
   - DOAJ（オープンアクセス）

3. **特定の分野を網羅的に検索**:
   - コンピュータサイエンス: arXiv、ACM Digital Library、IEEE Xplore
   - 医学・生命科学: PubMed、PubMed Central
   - 物理学: arXiv、IOPscience

### 統合検索エンジンの利用

複数のデータベースを横断的に検索できるエンジンを利用することで、網羅的な検索が可能となります。

#### 1. OpenAlex（推奨）
- **URL**: https://openalex.org/
- **特徴**:
  - 209百万件の著作物、2013百万件の著者、124千件の出版物などを網羅
  - REST APIを通じて高ボリュームのデータ取得が可能
  - 完全にオープンで無料
- **網羅性**: ⭐⭐⭐⭐⭐（非常に高い）

#### 2. BASE（Bielefeld Academic Search Engine）
- **URL**: https://www.base-search.net/
- **特徴**:
  - 多数の学術リポジトリからデータを集約
  - オープンアクセスの論文を中心に検索
- **網羅性**: ⭐⭐⭐⭐（高い）

#### 3. CORE
- **URL**: https://core.ac.uk/
- **特徴**:
  - オープンアクセスの論文を集約
  - APIを提供
- **網羅性**: ⭐⭐⭐⭐（高い）

#### 4. Semantic Scholar
- **URL**: https://www.semanticscholar.org/
- **特徴**:
  - AIを活用した論文検索エンジン
  - 関連性の高い論文を見つけやすい
  - APIを提供
- **網羅性**: ⭐⭐⭐⭐（高い）

### APIを統合した網羅的な検索の実装

```typescript
// 複数のデータベースから論文を検索する関数
const searchComprehensive = async (keyword: string) => {
  const results = await Promise.allSettled([
    // OpenAlexから引用数の多い論文を検索
    fetch(`https://api.openalex.org/works?search=${keyword}&sort=cited_by_count:desc&per_page=10`)
      .then(res => res.json())
      .then(data => data.results.map(paper => ({
        source: 'OpenAlex',
        title: paper.title,
        authors: paper.authorships?.map(a => a.author.display_name) || [],
        citationCount: paper.cited_by_count || 0,
        publicationDate: paper.publication_date,
        url: paper.primary_location?.landing_page_url || paper.doi || null,
        abstract: paper.abstract || null
      }))),
    
    // arXivから最新の論文を検索
    fetch(`http://export.arxiv.org/api/query?search_query=all:${keyword}&sortBy=submittedDate&sortOrder=descending&start=0&max_results=10`)
      .then(res => res.text())
      .then(xml => {
        // XMLをパースして論文情報を取得
        // （実装例は省略）
        return []
      }),
    
    // CiNiiから論文を検索
    fetch(`https://ci.nii.ac.jp/opensearch/search?q=${keyword}&format=json&count=10`)
      .then(res => res.json())
      .then(data => {
        // JSONをパースして論文情報を取得
        // （実装例は省略）
        return []
      })
  ])
  
  // 結果を統合して、引用数でソート
  const allPapers = results
    .filter(r => r.status === 'fulfilled')
    .flatMap(r => r.value)
    .sort((a, b) => (b.citationCount || 0) - (a.citationCount || 0))
  
  return allPapers
}
```

### 網羅的な検索の注意点

1. **APIの制限**: 各データベースのAPIには利用制限がある
2. **データの重複**: 複数のデータベースに同じ論文が含まれる場合がある
3. **データの品質**: 各データベースのデータ品質が異なる
4. **レスポンス時間**: 複数のAPIを呼び出すため、レスポンス時間が長くなる可能性がある

## 10. まとめ

### 論文にアクセスできる主要なサイト
1. **J-STAGE**: 日本の学術論文（日本語、オープンアクセスが多い）
2. **arXiv**: 国際的なプレプリント（英語、オープンアクセス）
3. **PubMed**: 医学・生命科学（英語）
4. **Google Scholar**: 検索エンジン（様々なデータベースから検索）

### Google Scholarの使用について
- ✅ **個人利用（読むだけ）**: 問題なし
- ⚠️ **商用利用**: 各論文のライセンスを確認する必要がある
- ❌ **API**: 公式APIは提供されていない
- ✅ **J-STAGEとCiNiiも検索対象**: Google ScholarはJ-STAGEやCiNiiの論文も検索できるため、日本の学術論文を広く検索可能

### 速読アプリでの推奨
- **J-STAGE**: 日本語の論文が豊富で、オープンアクセスが多い（直接APIを使用）
- **arXiv**: 英語の論文も対応できる（APIが提供されている）
- **Google Scholar**: 検索エンジンとして使用（ただし、APIがないため実装が難しい。ただし、J-STAGEやCiNiiの論文も検索対象に含まれている）
